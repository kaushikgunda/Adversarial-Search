Problems with MiniMax

Let’s see how many nodes we have to traverse in order to decide an action in our game of tic-tac-toe. Let us calculate it to decide the first move.Possible Actions = 9,Moves=9. First Level: 9 Nodes. Second Level: 8 available positions to move.So 9*8=72 nodes in this level. ..Final Level: 9*8…*1=9! Nodes. So total nodes to iterate will be in of the order of 10^6. Though a normal computer can simulate it in a very few seconds we have to understand that tic-tac-toe is a very simple game.Games like chess have many many times over the number of moves and actions.
See for yourself the order how the number of nodes to iterate changes with the game configuration.

Alpha-beta

Formalising the idea,let’s define two parameters Alpha and Beta.
Alpha- maximum lower bound of possible solutions
Beta- minimum upper bound of possible solutions
So following above logic whenever beta <= alpha we can prune the rest of nodes. Just like the visualisations for minimax we can represent alpha- beta search also with each state having [alpha,beta] that keeps updating .
